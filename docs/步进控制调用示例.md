# 比赛步进控制板调用示例

控制板固定串口为 `/dev/stepper_controller`，使用 115200 波特率。STM32 固件协议为 v1.0，支持两个轴：`axis: 0`（YL，PA0/PA1）和 `axis: 1`（YR，PA2/PA3）。

> 安全规则：首次上电、STM32 重启、串口断开重连、机构被手动推动或堵转后，理论位置均不可信。此时只能做小幅相对移动；使用绝对位置前，必须重新人工设零。

> 龙门规则：两台步进电机共同驱动同一个龙门架时，正式比赛只调用 `move_gantry` 和 `set_gantry_zero`。默认配置已经拒绝单独 `move_axis`/`set_zero`，以避免一侧单独运行导致龙门架跑斜、卡死。

## 2. 不转动电机的通信检查

这一步只查询 STM32 固件信息，不输出 STEP 脉冲：

```bash
python3 - <<'PY'
from wuliusai_stepper_controller.protocol import StepperSerial

board = StepperSerial('/dev/stepper_controller')
try:
    print('STM32 info:', board.connect())
finally:
    board.close()
PY
```

最新固件预期结果：`STM32 info: (1, 1, 2)`，分别表示协议主版本 1、次版本 1、两路轴。若仍显示 `(1, 0, 2)`，说明尚未烧录支持龙门同步命令的最新固件。

## 3. 原始脉冲低速测试

先让机构留出安全余量。`steps` 是驱动器收到的 STEP 脉冲数，尚未换算成毫米或角度。

```bash
# axis 0 正向 100 脉冲；低速空载测试
ros2 run wuliusai_stepper_controller stepper_serial_test \
  --axis 0 --steps 100 --pps 200 --accel 500

# axis 0 反向退回 100 脉冲
ros2 run wuliusai_stepper_controller stepper_serial_test \
  --axis 0 --steps -100 --pps 200 --accel 500

# axis 1 正向 100 脉冲
ros2 run wuliusai_stepper_controller stepper_serial_test \
  --axis 1 --steps 100 --pps 200 --accel 500
```

若方向与机构语义相反，后续修改 YAML 中对应轴的 `direction: 1` 为 `direction: -1`，不要改变 STM32 接线或在上层把目标值取反。

## 4. 标定毫米/角度

对直线轴，给定较大的安全脉冲数 `N`，量取实际行程 `D mm`：

```text
pulses_per_unit = N / D
```

例如 1600 脉冲实际移动 40 mm，则 `pulses_per_unit: 40.0`。旋转轴同理，若 3200 脉冲旋转 180 度，则设为 `17.777778`，并把该轴 `unit` 改为 `deg`。

把结果写入：

```text
/home/rog/ros2_ws/wuliusai/ros2_ws/src/wuliusai_stepper_controller/config/stepper_controller.yaml
```

同时设置实际机械安全范围，例如：

```yaml
axis_0.min_position: 0.0
axis_0.max_position: 250.0
axis_0.max_speed: 20.0
axis_0.acceleration: 50.0
```

修改 YAML 后重新构建并重新 source：

```bash
cd /home/rog/ros2_ws/wuliusai/ros2_ws
colcon build --symlink-install
source install/setup.bash
```

## 6. 人工设零

### 龙门整体设零（正式使用）

将龙门架人工移动到共同机械零点、确认两侧无歪斜后，一次设定两侧：

```bash
ros2 service call /stepper_controller/set_gantry_zero \
  wuliusai_stepper_msgs/srv/SetGantryZero "{position: 0.0}"
```

## 7. ROS2 Action 调用

### 龙门同步移动（比赛正式接口）

该 Action 将同一物理目标转换为两侧各自的脉冲数，并通过一条 STM32 同步命令启动两路电机。`max_speed` 是龙门的物理单位速度；`0.0` 使用 YAML 中两侧共同允许的保守速度。

```bash
# 从人工零点同步移动到 120 mm
ros2 action send_goal /stepper_controller/move_gantry \
  wuliusai_stepper_msgs/action/MoveGantry \
  "{relative: false, target: 120.0, max_speed: 15.0}" --feedback

# 在当前位置基础上，同步前进 5 mm
ros2 action send_goal /stepper_controller/move_gantry \
  wuliusai_stepper_msgs/action/MoveGantry \
  "{relative: true, target: 5.0, max_speed: 0.0}" --feedback
```

动作被取消、超时、单侧停止或 STM32 报错时，节点会停止两侧并把两轴标记为“未校零”；再次绝对移动前必须重新人工设零。

### 独立轴调试接口

`move_axis` / `set_zero` 仍保留给未来独立机构调试。当前龙门配置默认禁用它们；不要为了让两侧同步而在不同终端分别调用两次 `move_axis`。

## 8. 停止

```bash
# 只停止 axis 0
ros2 service call /stepper_controller/stop \
  wuliusai_stepper_msgs/srv/StopAxis "{axis: 0}"

# 停止全部轴；255 表示全部轴
ros2 service call /stepper_controller/stop \
  wuliusai_stepper_msgs/srv/StopAxis "{axis: 255}"
```

停止后请检查位置；若机构可能发生丢步或碰撞，重新人工设零后再进行绝对运动。

## 9. Python 调用骨架

比赛状态机应使用 ROS2 Action Client，而不是绕过 ROS2 直接写串口：

```python
from rclpy.action import ActionClient
from wuliusai_stepper_msgs.action import MoveGantry

self.move_client = ActionClient(self, MoveGantry, '/stepper_controller/move_gantry')
goal = MoveGantry.Goal()
goal.relative = False
goal.target = 120.0
goal.max_speed = 15.0

await self.move_client.wait_for_server()
goal_handle = await self.move_client.send_goal_async(goal)
result = await goal_handle.get_result_async()
if not result.result.success:
    raise RuntimeError(result.result.message)
```

只有 `result.result.success` 为真，比赛状态机才应进入下一步动作。
